1.
Система контроля версий (VCS) — это программное обеспечение, позволяющее отслеживать изменения в файлах проекта, хранить их версии, возвращаться к предыдущим состояниям, сливать изменения от разных авторов и координировать совместную работу над кодом.
GitHub — это веб платформа для хостинга и совместной разработки IT проектов на базе системы контроля версий Git. Предоставляет инструменты для:
Репозиторий — это хранилище файлов проекта вместе с полной историей их изменений.
Коммит (commit) — это зафиксированное состояние проекта в определённый момент времени. 
2. Локальный vs удалённый репозиторий
Локальный репозиторий
находится на жёстком диске вашего компьютера;
вы работаете с ним напрямую (добавляете файлы, делаете коммиты, переключаете ветки);
не требует интернет соединения;
ваши изменения видны только вам, пока вы не отправите их на сервер;
служит «рабочей зоной» для экспериментов и промежуточной разработки.
Удалённый репозиторий (на GitHub)
хранится на сервере GitHub (или другом хостинге);
доступен команде через интернет;
используется для:
резервного копирования кода;
совместной работы (pull requests, обзоры кода);
развёртывания и CI/CD;
публикации открытого ПО;
3.
Коммит хранит полное состояние всех отслеживаемых файлов на момент фиксации, а не только диффы (разницы) между версиями. Это позволяет:
мгновенно восстановить проект в точном виде, как он был на момент коммита;
избежать накопления цепочек патчей, которые могут конфликтовать или теряться;
гарантировать целостность проекта на каждую версию.
При этом Git оптимизирует хранение: физически дубликаты файлов не копируются, но логическая модель — это именно снимки, а не список изменений.
Каждый коммит должен содержать:
1.	Уникальный идентификатор (хеш)
2.	Список изменённых файлов
3.	Метаданные автора
4.	Сообщение (commit message) 
5.	Ссылка на родительский коммит
